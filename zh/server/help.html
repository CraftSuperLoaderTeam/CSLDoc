# 开服帮助

# 关于服务端优化

因CSL本质基于SpigotMC开发,所以优化方案基本与SpigotMC无太大出入,您只需要依照市面上一些对Java应用程序或Java版Minecraft服务端的优化思路即可.对于如何优化,我们给出了以下解决方案

  • 硬件加持
  • JavaVM
  • MCBBS-详细服务端调优方式 (opens new window)

# 硬件优化

# CPU

与一些Web程序不同,虽然MinecraftServer涉及网络I/O,但其性能优化点主要还是在数据处理性能上,而因核心实现原因,MinecraftServer会非常注重CPU的单核性能.所以选购服务器或CPU时我们建议您选择单核性能较为强劲的配置

# 内存

跟大部分Java应用程序一样,MinecraftServer也是非常占用内存的,如果您想搭建一台玩家超过10人以上甚至更多的服务器,那么我们建议您对于内存的容量要有一个好的选择

ECC内存完全没有必要,这个只用于一些对数据准确率要求非常高的服务器,ECC内存一大弊端就是读写速度比DDR内存要慢不少

# 硬盘

如果您打算运营一台服务器而不是简单的与朋友联机,那么大容量的硬盘或另外准备一台数据库是一个不错的选择

提示

一定要经常备份您的数据,没有人能保证数据永远不会丢失,你要做的就是防范任何突发情况

# 虚拟机优化

关于虚拟机的优化,在阅读本栏目之前我们推荐您阅读一篇出自MCBBS的帖子,其作者速食冻橘详细的介绍了您如何选择适合您设备的JVM类型,在这里感谢一下速食冻橘花费大量时间和精力为用户们提供较为准确的测试结果

以下是CSL开发组自行开发中得出的性能优化结论,不保证百分百正确,仅供参考

# Hotspot JVM

由Oracle官方发布的标准JVM,其性能在解释性语言领域也占有一席之地.

好像图片有点糊,截图时候没有处理,详细信息查看下面说明罢(悲)

您可能注意到版本号是1_12_R1/v0.0.8,因为开发中并没有更改版本号,但是功能都是1_12_R1/v0.0.9的

  • 配置

CPU: I5-10400F 6核心12线程
RAM: 32GB(16+16) DDR4 2666MHz
ROM: 240GB SSD

  • 运行数据

MinecraftVersion: 1.12.2
OpenJDK: x64 Java8u301
CSLVersion: 1_12_R1/v0.0.9
CPU利用率: 14.1%
内存使用: 488.5MB

以上测试信息为服务端启动时的数据,装载了两款JS插件和两款Bukkit插件,一共花费了2.068s服务端启动完毕

  • 2秒对于服务端启动还是比较快的(XIAOYI12个人认为)
  • 对于内存充足的用户来说,您只需要注意一下JVM执行效率即可(但能做内存优化还是做,如果玩家人数多了的话再大的内存不经过调优还是会炸)

# GraalVM

由Oracle官方发布的新一代JVM,相比于Hotspot有了缩短了启动时间,并对CPU和内存做了优化

以下就不放图了,感觉CPU利用率不太稳定,测试结果不准确

  • 配置

CPU: I5-10400F 6核心12线程
RAM: 32GB(16+16) DDR4 2666MHz
ROM: 240GB SSD

  • 运行数据

MinecraftVersion: 1.12.2
OpenJDK GraalVM: x64 Java17
CSLVersion: 1_12_R1/v0.0.9
CPU利用率: 11.4%
内存使用: 344.5MB

以上测试信息为服务端启动时的数据,装载了两款JS插件和两款Bukkit插件,一共花费了2.049s服务端启动完毕

  • 启动时间好像没有什么显著优化啊
  • GraalVM仅建议用于编译成平台可执行文件时使用,平时使用非常不稳定.
  • 对于将MinecraftServer编译成平台可执行文件其手续非常复杂,需要监控代理,以及各种Maven插件调试等,因GraalVM编译出来的平台可执行文件目前对JAVA反射支持还不太成熟,XIAOYI12推断Bukkit的事件系统会瘫痪.从而导致服务端无法正常运行插件

速食冻橘也指出GraalVM的native-image无法编译MinecraftServer
